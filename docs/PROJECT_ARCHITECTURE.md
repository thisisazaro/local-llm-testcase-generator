# Архитектура прототипа

## Цель
Локальный информационный сервис для генерации тест-кейсов по документам аналитики (PDF) без обращения к внешним API.

## Контур обработки
1. `POST /upload`: загрузка PDF.
2. Ingestion: извлечение текста, чанкинг, индексирование в Chroma.
3. Retrieval: отбор релевантных фрагментов по запросу пользователя.
4. `POST /generate`: формирование prompt + вызов локальной LLM.
5. Постобработка: извлечение JSON, нормализация, дедупликация и валидация тест-кейсов.
6. `POST /evaluate`: расчёт метрик качества для generated/reference наборов.

## Компоненты
- `app/main.py`: API-слой FastAPI.
- `app/ingest.py`: подготовка входного документа.
- `app/vector_store.py`: Chroma + эмбеддинги.
- `app/generator.py`: интеграция с Ollama/vLLM и JSON-парсинг.
- `app/prompts.py`: системный и пользовательский prompt.
- `app/config.py`: централизованные настройки окружения.

## Локальные провайдеры LLM
- `LLM_PROVIDER=ollama`: вызов `OLLAMA_BASE_URL/api/generate`.
- `LLM_PROVIDER=openai_compatible`: вызов `OPENAI_COMPAT_BASE_URL/chat/completions` (например, vLLM).
- `LLM_PROVIDER=mock`: безопасный режим демонстрации без GPU.

## Почему это подходит для защиты
- Полный E2E pipeline: документ -> retrieval -> генерация -> структурированный результат.
- Управление генерацией через `user_prompt` и параметры запроса.
- Готовый контур эксперимента и метрик качества для доказательной части.
