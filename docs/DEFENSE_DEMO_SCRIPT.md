# Сценарий демонстрации на защите

## 1. Постановка задачи (1-2 минуты)
- Проблема: много ручной работы QA при подготовке тест-кейсов по аналитике.
- Цель: локальный сервис генерации тест-кейсов без внешних API.

## 2. Демонстрация прототипа (4-6 минут)
1. Показать `GET /health` и параметры локальной модели.
2. Загрузить PDF через `POST /upload`, получить `file_id`.
3. Вызвать `POST /generate` с кастомным `user_prompt`.
4. Показать структурированный JSON тест-кейсов и `source_ref`.

## 3. Эксперимент (3-4 минуты)
1. Показать датасет сценариев эксперимента.
2. Запустить `evaluation/run_experiment.py`.
3. Открыть `evaluation/out/report.md` и интерпретировать метрики.

## 4. Выводы и ограничения (1-2 минуты)
- Сервис ускоряет подготовку тест-кейсов.
- Критично сохранять QA-ревью как контроль качества.
- Дальнейший рост качества: дообучение LoRA и улучшение эмбеддингов.
